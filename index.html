给 UIKit 开发者的 Flutter 指南

这份指南旨在帮助 UIKit 开发者将现有的 iOS 开发经验应用到 Flutter 开发中。

视图与组件（Views vs. Widgets）

  • Flutter 中的 Widget：Flutter 中 Widget 是构建 UI 的基本单元，与 UIKit 中的 UIView 类似，但有本质区别。Widget 是不可变的，当需要更新 UI 时，Flutter 会创建新的 Widget 实例树。
  • StatelessWidget 与 StatefulWidget：StatelessWidget 是没有状态的，一旦构建就不会改变；StatefulWidget 则包含状态，可以在运行时更新。例如，一个不变的 logo 可以用 StatelessWidget 表示，而需要根据 HTTP 请求动态更新的 UI 则需用 StatefulWidget。

更新 Widget

  • 状态管理：在 Flutter 中，通过改变 Widget 的状态来更新 UI。StatefulWidget 的状态对象（State）会在 Widget 树重建时保留，不会丢失。

布局（Layout）

  • 布局声明：在 Flutter 中，通过编写代码来声明布局，构建 Widget 树。例如，使用 Scaffold、AppBar 和 Center 等 Widget 来构建界面。
  • 动态更新布局：可以通过控制父 Widget 返回的子 Widget 来动态添加或移除子 Widget，这与 UIKit 中的 addSubview 和 removeFromSuperview 不同。

动画（Animations）

  • 动画库：Flutter 提供了动画库，使用 AnimationController 来控制动画的播放、暂停等。例如，使用 CurvedAnimation 可以实现沿着特定曲线的动画效果。

屏幕绘制（Drawing on the screen）

  • CustomPaint 与 CustomPainter：Flutter 提供了基于 Canvas 类的 API，通过 CustomPaint 和 CustomPainter 来绘制图形。例如，实现一个签名绘制功能。

透明度（Widget opacity）

  • Opacity Widget：在 Flutter 中，通常需要使用 Opacity Widget 来设置 Widget 的透明度，与 UIKit 中的.alpha 属性类似。

自定义 Widget（Custom Widgets）

  • 组合而非继承：在 Flutter 中，通过组合小的 Widget 来构建自定义 Widget，而不是通过继承。例如，创建一个包含标签的 CustomButton，而不是继承 ElevatedButton。

依赖管理（Managing dependencies）

  • Pubspec.yaml：Flutter 使用 Dart 的构建系统和 Pub 包管理器来处理依赖。通过 pubspec.yaml 文件来声明外部依赖，而不是使用 CocoaPods。

导航（Navigation）

  • Navigator 与 Routes：Flutter 中使用 Navigator 和 Routes 来管理页面导航，类似于 UIKit 中的 UINavigationController。可以通过 push 和 pop 方法来导航到不同的页面。

多语言支持（Localization）

  • flutter_localizations 包：Flutter 使用 flutter_localizations 包来支持多语言，需要在 app widget 中指定 localizationsDelegates 和支持的 locales。

视图控制器（ViewControllers）

  • WidgetsBinding 观察者：Flutter 中没有 ViewController 的概念，但可以通过 WidgetsBinding 观察者来监听应用的生命周期事件，如 inactive、paused、resumed 等。

列表视图（List view）

  • ListView：Flutter 中使用 ListView 来显示列表，可以通过 ListView.builder 来高效地构建动态列表或大数据量的列表。

事件处理（Gesture detection and touch event handling）

  • GestureDetector：Flutter 中使用 GestureDetector 来处理各种手势事件，如点击、双击、长按、拖动等。

主题与样式（Themes, styles, and media）

  • MaterialApp 与 Cupertino：Flutter 提供了 Material Design 和 Cupertino 风格的组件库，以适应不同的设计语言。可以通过 ThemeData 来定制主题颜色和样式。

字体与图片（Fonts and images）

  • 自定义字体：在 Flutter 中，通过在 pubspec.yaml 中声明字体文件来使用自定义字体。
  • 图片资源：图片和其他资源作为资产（assets）管理，通过 AssetBundle 来访问。

表单输入（Form input）

  • TextEditingController：在 Flutter 中，使用 TextEditingController 来获取用户输入，可以通过 InputDecoration 来设置提示文本和错误信息。

异步编程（Threading & asynchronicity）

  • Dart 的异步模型：Dart 是单线程的，但支持异步编程。使用 async/await 来处理网络请求等异步操作，避免 UI 卡顿。对于计算密集型任务，可以使用 Isolate 来利用多核 CPU。

网络请求（Making network requests）

  • http 包：Flutter 中使用 http 包来简化网络请求，通过 await http.get() 等方法来获取数据。

进度指示（Progress indication）

  • ProgressIndicator：在执行长时间任务时，使用 ProgressIndicator 来显示进度，通过布尔标志来控制其显示和隐藏。
